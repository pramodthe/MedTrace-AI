# Implementation Plan: Google Stitch-Inspired Voice-to-Report Dashboard

We will completely transform the Voice feature into an enterprise-grade **Voice-to-Report Dashboard** inspired by Google Stitch.

Rather than real-time continuous streaming, the user will record their audio first. Upon completion, the audio is saved to a backend database, transcribed via Whisper, and processed by LangGraph to generate a structured report.

---

## Proposed Changes

### 1. 🐍 Backend Database & API (`/backend`)

We will create a lightweight local SQLite database to store recordings, transcriptions, and generated reports, enabling full persistence and playback.

#### [NEW] [database.py](file:///Users/pramodthebe/Desktop/predictive_state_updates/backend/database.py)
- Setup a SQLite database using `sqlite3` to store:
  - `id` (UUID primary key)
  - `timestamp` (ISO datetime)
  - `duration` (formatted duration e.g. "0:45")
  - `transcript` (text from Whisper)
  - `report` (markdown text from LangGraph)
  - `audio_base64` (raw audio payload for native playback of any historical session)

#### [MODIFY] [main.py](file:///Users/pramodthebe/Desktop/predictive_state_updates/backend/main.py)
- Create endpoints:
  - `GET /api/sessions`: Fetch all saved recordings from the database.
  - `POST /api/sessions`: Save a new recording (accepts base64 audio and duration), transcribes it using Whisper, executes the LangGraph agent to generate the report, persists everything to SQLite, and returns the saved session.
